git 명령어

  • git init

    git 선언

  • git status

    git 변화 감지

  • git add .

    git index영역으로 이동

  • git commit -m "commit log"

    git commit "commit log"

  • git log

    git log 확인

  • git reset --soft 돌아가고자하는 log의 해쉬값

    git commit log 명 변경

  • git reset --mixed 돌아가고자하는 log의 해쉬값

    git 파일 내용 수정/ index영역 삭제

  • git reset --hard 돌아가고자하는 log의 해쉬값

    git 파일 삭제

  • git reflog

    지워진 log까지 보여줌

  • git commit -ammend -m "commit log"

    마지막 commit log 명 변경

  • touch "파일명"

    파일 생성

  • touch "파일명"

    파일 생성

  • git branch

    branch 확인

  • git branch branch명

    branch 생성

  • git checkout branch명

    branch로 이동

  • git checkout -b branch명

    branch 생성 후 이동

  • git merge branch명

    branch명를 현재 branch로 merge

  • git rebase -i head ~n

    head에서 n번째 log까지 정리

  • git rebase -i head ~n

    d : 삭제/ pick : 선택/ r : 최상단의 log 변경/ s : 가장 과거의 log(pick)를 제외한 log 삭제(s)

  • git remote add origin https://

    클라우드 저장소 연결

  • git remote -v/ git ls-remote

    클라우드 저장소 연결 확인

  • git push origin main

    클라우드 저장소의 main branch로 업로드 및 병합

  • git push origin new branch

    클라우드 저장소에 새로운 branch 생성 후 업로드

  • git remote rm origin

    잘못된 주소 연결 해제

  • git pull origin main

    클라우드 저장소의 main branch 내용 다운로드 및 병합

  • git clone https://

    클라우드 저장소에서 main branch 내용 가져옴

  • git fetch orgin

    클라우드 저장소의 branch를 가져옴/ merge x

  • git checkout -b branch origin/branch

    branch를 만들고 orgin/branch의 내용을 업로드 및 병합

  • git merge --squash branch명

    branch명의 모든 log를 하나의 log로 정리해서 현재 branch로 merge

  • git merge --no-ff branch명

    merge 시 log명 생성/(fast-foward-merge 시 log가 남지 않는다)

  • git tag tag명

    현재 branch에 tag를 남김

  • git push --tags orgin main

    클라우드 저장소에 tag를 남기면서 업로드

  • git tag -n

    입력된 tag 값 확인
